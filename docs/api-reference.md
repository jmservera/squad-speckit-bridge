# API Reference

Complete reference for the Squad-SpecKit Bridge CLI and programmatic interfaces.

## CLI Commands

### Global Options

All commands support:

```bash
--config <path>          Path to bridge.config.json (default: ./bridge.config.json)
--verbose               Verbose output (default: false)
--squad-path <path>     Path to Squad directory (default: .squad/)
--speckit-path <path>   Path to Spec Kit directory (default: .specify/)
--help                  Show command help
--version               Show version
```

---

## `squad-speckit-bridge init`

Initialize the bridge in your project.

### Usage

```bash
npx @jmservera/squad-speckit-bridge init [options]
```

### Options

```
--squad-path <path>         Path to Squad directory (default: .squad/)
--speckit-path <path>       Path to Spec Kit directory (default: .specify/)
--config-file <path>        Config file location (default: bridge.config.json)
--auto-workflow             Set up GitHub Actions workflow (default: false)
--install-path <path>       Where to copy plugins (default: project root)
--no-verify                 Skip verification after install (default: false)
```

### Output

On success:
```
✓ Squad initialized
✓ Spec Kit initialized
✓ Bridge plugin installed to .squad/skills/speckit-bridge/
✓ Bridge extension installed to .specify/extensions/squad-bridge/
✓ Configuration created: bridge.config.json
✓ GitHub workflow created: .github/workflows/squad-bridge-review.yml
✓ All checks passed
```

### Exit Codes

- `0` — Success
- `1` — Squad or Spec Kit not initialized
- `2` — Configuration error
- `3` — File system error (permissions, etc.)

## `squad-speckit-bridge context`

Inject Squad memory before Spec Kit planning.

### Usage

```bash
npx @jmservera/squad-speckit-bridge context [options]
```

### Options

```
--spec-dir <path>           Spec Kit directory (default: .specify/)
--output <path>             Output file for context (default: squad-context.md)
--budget <tokens>           Context budget in tokens (default: from config)
--max-histories <num>       Max agent histories to include (default: 3)
--prioritize-recent         Prioritize recent learnings (default: true)
--format <json|markdown>    Output format (default: markdown)
--since <date>              Include learnings since date (ISO 8601)
--filter <pattern>          Include only learnings matching pattern
--dry-run                   Show what would be generated without writing
```

- GitHub Integration: Progressive adoption

Total learnings available: 47
Included in context: 12
Excluded (budget): 35

## `squad-speckit-bridge review <task-file>`

Run design review ceremony for Spec Kit tasks.

### Usage

```bash
npx @jmservera/squad-speckit-bridge review <path-to-tasks.md> [options]
```

### Arguments

```
<task-file>    Path to tasks.md generated by Spec Kit (required)
```

### Options

```
--interactive              Interactive review mode (default: true)
--output <path>           Save reviewed tasks to file (default: <name>-reviewed.md)
--focus <pattern>         Review only tasks matching pattern
--skip-tasks <list>       Skip tasks by ID (comma-separated)
--auto-approve            Approve all tasks without review
--format <json|markdown>  Output format (default: markdown)
--detailed                Show detailed context for each task
```

### Interactive Mode Commands

```
(a) Approve task               - Accept task as-is
(m) Add note                   - Add team note to task
(f) Flag concern               - Mark task as risky/concerning
(e) Edit task                  - Edit task description/criteria
(s) Skip task                  - Review later
(p) Previous task              - Go to previous task
(n) Next task                  - Go to next task
(?) Show context               - Show squad-context.md excerpt
(q) Quit                       - Exit review
```

### Output (Markdown Format)

```markdown
# Tasks Review: Feature 032

Review date: 2026-03-24
Reviewed by: squad-speckit-bridge
Status: Approved (8/8 tasks)

## Task 1: Extract shared auth middleware
- Status: ✓ Approved
- Effort: 3 points
- Dependencies: Task 2

## Task 2: Add rate limiting to API routes
- Status: ⚠ Flagged
- Effort: 5 points
- Team note: "Check our middleware order learnings"
- Dependencies: Task 1

[... more tasks ...]

## Review Summary
- Total tasks: 8
- Approved: 7
- Flagged: 1
- Notes added: 2
```

### Exit Codes

- `0` — Review completed
- `1` — File not found or invalid format
- `2` — User cancelled review
- `3` — Configuration error

## `squad-speckit-bridge issues <task-file>`

Create GitHub issues from reviewed tasks.

### Usage

```bash
npx @jmservera/squad-speckit-bridge issues <path-to-tasks-reviewed.md> [options]
```

### Arguments

```
<task-file>    Path to tasks-reviewed.md from design review (required)
```

### Options

```
--dry-run                   Show what would be created without creating
--auto-assign               Assign issues automatically (default: true)
--assignment-strategy       round-robin|random|none (default: from config)
--assignee-pool <list>      Agents to assign to (comma-separated)
--labels <label1,label2>    Additional labels to apply
--label-prefix <prefix>     Prefix for auto-generated labels (default: "squad:")
--milestone <title>         Add to GitHub milestone
--base-branch <branch>      Base branch for PRs (if supported)
--body-template <path>      Custom issue body template (Handlebars)
```

### Output (Success)

```
✓ Created 8 issues from 8 tasks

Issues created:
  #1234 Extract shared auth middleware
  #1235 Add rate limiting to API routes
  #1236 [FLAGGED] Investigate middleware ordering
  ...

Assignment:
  Ralph: 3 issues
  Jordan: 2 issues
  Morgan: 2 issues
  (unassigned): 1 issue

Labels applied: squad, auth, refactor
Milestone: v1.2.0

Issues ready for pickup. Use `gh pr list` to track progress.
```

### Dry-Run Output

```
Would create 8 issues:

Issue 1: Extract shared auth middleware
  Effort: 3 points
  Assignee: ralph
  Labels: squad, auth, refactor
  Body: [issue body preview]
  
... (more issues)

Total estimated effort: 28 points
Estimated timeline: ~2 weeks (at 15 pts/week velocity)

(use --no-dry-run to create)
```

### Exit Codes

- `0` — Issues created successfully
- `1` — File not found or invalid format
- `2` — GitHub API error
- `3` — Configuration or validation error

## `squad-speckit-bridge sync`

Sync execution learnings back to Squad.

### Usage

```bash
npx @jmservera/squad-speckit-bridge sync [options]
```

### Options

```
--since <date>              Sync learnings since date (ISO 8601)
--closed-issues             Sync from closed GitHub issues only (default: true)
--all-issues                Sync from all issues (overrides --closed-issues)
--output <path>             Save sync report to file
--format <json|markdown>    Output format (default: markdown)
--dry-run                   Show what would be synced without committing
--labels <label1,label2>    Only sync issues with these labels
```

### Output (Markdown Format)

```markdown
# Squad Learning Sync Report

Sync date: 2026-03-24T14:30:00Z
Period: 2026-03-17 to 2026-03-24

## Issues Processed

Total: 8
Closed: 8
In progress: 0
Skipped: 0

## Learnings Extracted

### Performance & Scalability
- Database query optimization resulted in 3x improvement
- Connection pooling is essential at load
- N+1 queries must be caught in tests

### Development Patterns
- Error handling pattern worked well
- Middleware ordering is critical (reconfirmed)

### Team Velocity
- Authentication tasks took 50% longer than estimated
- Rate limiting was simpler than expected

## Integration Points

Ready for next planning cycle:
- squad-context.md can now reference these learnings
- Spec Kit planning will include execution patterns

Suggested next focus areas:
- Database optimization (proven valuable)
- Error handling standards (team consensus)
- Task estimation refinement (velocity feedback)
```

### Output (JSON Format)

```json
{
  "syncDate": "2026-03-24T14:30:00Z",
  "period": {
    "start": "2026-03-17",
    "end": "2026-03-24"
  },
  "issuesProcessed": 8,
  "learningsExtracted": [
    {
      "category": "Performance",
      "summary": "Database query optimization resulted in 3x improvement",
      "source": "#1234",
      "agent": "ralph",
      "confidence": "high"
    }
  ],
  "nextPlanningContext": "squad-context.md updated with 12 new learnings"
}
```

### Exit Codes

- `0` — Sync completed successfully
- `1` — No issues found or nothing to sync
- `2` — GitHub API error
- `3` — Configuration or file I/O error
