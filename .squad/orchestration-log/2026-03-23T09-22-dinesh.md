## Task: MCP Server Technical Feasibility & Integration Approaches

**Assignment:** Deep technical assessment of MCP server approach, Squad plugin capabilities, Spec Kit extension hooks, and standalone CLI. Output: implementation estimates, LOC projections, architectural blueprints for each approach.

**Approach:**
- Surveyed MCP server: stdio transport design, tool surface, registration, ~500 LOC estimate
- Surveyed Squad plugin: SKILL.md structure, knowledge-only model, no executable code path
- Surveyed Spec Kit extension: hooks available (after_tasks ✅, before_specify ❌), bash script execution model
- Surveyed standalone CLI: command design, ~760 LOC, tool discovery problem
- Built technical comparison table across 11 dimensions

**Key Findings:**

1. **MCP Server Feasibility: Confirmed, ~500 LOC**
   - Transport: stdio subprocess (same as GitHub MCP, Playwright MCP)
   - Tools: bridge_squad_context, read_speckit_tasks, create_issues_from_tasks, sync_learnings, check_integration_status
   - Setup: `npx squad-speckit-mcp@latest` in mcp-config.json
   - Limitations: passive (no auto-triggers), needs GitHub API token

2. **Squad Plugin: Knowledge-Only Layer**
   - SKILL.md: teachable, references MCP tools in metadata (no install/execute)
   - Ceremony definition: can add to ceremonies.md
   - Capability gap: no executable code, no file watching, no lifecycle hooks
   - Ready for v0.1

3. **Spec Kit Extension: Lifecycle Automation**
   - after_tasks hook: ✅ exists (fires after tasks.md generated)
   - after_implement: ✅ exists
   - before_commit, after_commit: ✅ exist
   - **Critical gap:** before_specify ❌, before_plan ❌
   - Workaround: manual command or before_tasks hook (late)
   - Ready for v0.1

4. **Standalone CLI: Not Recommended**
   - ~760 LOC, modest
   - Agent discoverability problem (no framework tool registration)
   - MCP server is superior for same complexity

**Phased Technical Recommendation:**
- **Phase 1 (NOW):** Squad plugin SKILL.md (~100 LOC) + Spec Kit extension with after_tasks hook (~50 LOC) + shared bridge script (~150 LOC) = **~300 LOC total**
  - Developer triggers bridge manually before planning (workaround)
  - Proves loop works, zero framework changes
  
- **Phase 2 (After loop proven):** MCP Server (~500 LOC)
  - Agents call tools directly for reliable parsing
  - Plugin + extension remain (knowledge + hooks)
  - Structured JSON replaces LLM-interpreted markdown

- **Phase 3 (If warranted):** Propose before_specify hook to Spec Kit
  - Full automation: bridge runs pre-planning
  - MCP + extension hooks = zero-manual-step loop

**Architecture Detail:**
- Hybrid package deploys to both frameworks from single source
- install.sh detects both frameworks, copies components correctly
- Bridge logic shared: summarize.js (Squad state → squad-context.md), parse-tasks.js (tasks.md → JSON)
