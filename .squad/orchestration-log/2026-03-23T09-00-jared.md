## Findings

### Critical Patterns

1. **Framework Weight Must Match Team Size**
   - Squad's 12-agent model created overhead for single developer
   - Spec Kit's zero-agent model was ideal for solo + Copilot
   - Need lightweight Squad mode (1-2 agents, no ceremonies)

2. **State Accumulation Is #1 Friction Point**
   - aithena's decisions.md reached 475KB — unusable as context
   - Both frameworks need automatic state pruning
   - Threshold-based archiving should be a framework feature

3. **Project Type Drives Framework Fit**
   - Greenfield → Spec Kit (fast spec-per-feature planning)
   - Brownfield → Squad (issue routing, milestone management)

4. **Spec-Per-Feature + Issue-Per-Task Are Complementary**
   - Combined model could use specs for features + issues for maintenance

5. **Constitutional Governance Scales Better**
   - Spec Kit: single 15KB constitution
   - Squad: distributed decisions across 5+ files
   - Squad needs a constitutional layer

6. **Progressive GitHub Integration**
   - sofia-cli: 0 Issues (too sparse)
   - aithena: 412 Issues (too dense)
   - Framework should support progressive adoption

## Decision Impact

Framework comparison criteria now clear: framework weight, state management, project-type fit. Validates that integration approach must handle state accumulation risk.

## Next Steps

- Implementation: Build state pruning for both frameworks
- Configuration: Lightweight Squad mode for small teams
